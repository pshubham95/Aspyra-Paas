FAQ

What is the Deploy and Test option?
The Deploy and Test option enables the user to select which packages he wishes have in their application environment. The user is then given access to a shell via the browser. This shell connects him to a container in which his desired packages have already been installed and his application folder will already be present in it. Now the user can run his application, make changes, test it etc. Everytime the system restarts the user has to manually restart services on container.
What is the Deploy option?
Using the Deploy option, a user can easily deploy his tested and ready website online. This option does not allow the user to make any changes to web application. It is a simple, fast and convenient way to deploy applications. Applications restart automatically when the system restarts.
What is the Develop and Deploy (CMS) option?
Using this option a user can easily develop a data publishing site using anyone of the following CMS choices - Wordpress, Drupal and Joomla. The installation procedure itself will happen on the college server and you can start using the application from there.
What is Calcy?
Calcy is a unique feature provided by us. It enables a user to enter some code he wishes to execute and that code will be run on the server and the result shall be displayed to the user. The user doesn't need to compile and run the code locally.
What is COE-Safe?
COE-Safe is a storage service which allows users to store data on the server. Users can store, view delete, edit, download and share files using this feature.
What is Linux Learning?
Linux learning is a service which helps users to learn more about using Linux based OS' using command line interface. This feature is basically useful for users new to Linux environment.
What is the deployment limit?
We have provided a limit of maximum 3 applications allowed per option (eg - 3 for CMS, 3 for Deploy etc.)
